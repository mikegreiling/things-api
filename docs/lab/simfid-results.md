# SIMFID — simulator-fidelity replay suite

**What it is.** SIMFID certifies [`src/write/vectors/simulator.ts`](../../src/write/vectors/simulator.ts) against the REAL Things app, op by op. For every simulator-covered op it compares two row-level DB deltas — the op replayed on the host through the full write pipeline with the simulator vector, and the same op driven through the guest CLI against the real app — after NORMALIZING away the differences that are not fidelity facts (uuids, wall-clock jitter, list positions), with declared TOLERANCES for the app's probe-proven nondeterminism. A residual difference is a **divergence the suite REPORTS** (a simulator bug OR newly-discovered app behaviour); SIMFID never edits the appliers. This is the automation of exactly the class of bug caught by hand on 2026-07-17 (the inbox-promotion filing semantics).

Spec: [probe-backlog §C](probe-backlog.md) (SIMFID). Laws + tolerances: [rsim-results.md](rsim-results.md). Wired into the drift runbook step 5 (re-certify after every golden rebuild).

## Suite architecture (in brief)

Two deltas per op, one normalizer, one comparator — all host-side and unit-tested ([`test/unit/simfid-normalize.test.ts`](../../test/unit/simfid-normalize.test.ts)).

- **SIM delta (host, real).** [`lab/simfid/replay.ts`](../../lab/simfid/replay.ts) drives each case ([`cases.ts`](../../lab/simfid/cases.ts)) through the SAME end-to-end path `test/engine/write-simulator.test.ts` exercises — guards → plan → execute → verified read-after-write → audit — against a fresh synthetic fixture, and snapshots the DB before/after ([`snapshot.ts`](../../lab/simfid/snapshot.ts)).
- **APP delta (clone, real).** [`lab/scripts/simfid.sh`](../../lab/scripts/simfid.sh) clones the golden, airgaps, pins the clock (2026-07-05, the RSIM clock), and drives each headless case's op through the guest CLI against the real app ([`guest-driver.py`](../../lab/simfid/guest-driver.py) + [`clone-manifest.json`](../../lab/simfid/clone-manifest.json)), collecting title-scoped before/after snapshots. The host ingests them into normalized deltas ([`ingest-clone.ts`](../../lab/simfid/ingest-clone.ts)).
- **Normalize** ([`normalize.ts`](../../lab/simfid/normalize.ts)): uuids → placeholders keyed by (kind, title, **container-aware** discovery order) — a template project and its instance are distinguished by their rule/template columns, children by their resolved container placeholder, so the same logical row lands on the same placeholder on BOTH sides even when titles collide (the make-repeating duplication is exactly this case). Wall-clock epochs (creation/modification/stop) → local-DATE buckets. List indexes → ranks.
- **Compare** ([`compare.ts`](../../lab/simfid/compare.ts)) + **tolerances** ([`tolerances.ts`](../../lab/simfid/tolerances.ts)): field-by-field; every difference is checked against the declared tolerances; verdict = MATCH / TOLERATED(&lt;which&gt;) / DIVERGENT(&lt;detail&gt;).

Run: `npm run lab:simfid` (report-only; `--gate` exits 1 on any DIVERGENT for the drift-runbook re-certification; `--filter <substr>` scopes cases). A fresh clone drive: `bash lab/scripts/simfid.sh`, whose per-case app deltas override the banked-evidence goldens for those cases.

## Declared tolerances

| Tolerance | What it absorbs | Evidence |
|---|---|---|
| `rt1-child-backlink` | On a subtree CHILD row, `rt1_repeatingTemplate` present on one side and absent on the other. The app stamps a per-child instance→template link on a NONDETERMINISTIC subset of children (6/7, 5/7, 6/6 across consecutive spawns of one series; headings never linked); the simulator emits plain children. | RSIM-R C2 / RSIM-S S1d |
| `instance-next-sentinel` | On an INSTANCE row, any `rt1_nextInstanceStartDate` value. The app leaves a junk `69760` sentinel; only the TEMPLATE's next-date drives generation; the simulator leaves it NULL. | RSIM |
| `index-rank` | A residual `index`/`todayIndex` rank difference (the app assigns real list positions, the simulator hardcodes 0). | SIMFID spec |
| `wallclock-bucket` | A date-bucket VALUE difference on `creationDate`/`userModificationDate`/`stopDate` **when both sides carry a timestamp** (the app backdates a minted instance to occurrence midnight; the simulator stamps write-time). **Presence-vs-absence is NOT tolerated** — one side holding a stop/creation date the other lacks is a structural fidelity fact the suite reports as DIVERGENT (tightened 2026-07-22, see the `todo-reopen` finding). | SIMFID spec |

The top-level instance row's OWN `rt1_repeatingTemplate` link is NOT tolerated (asserted) — only subtree children are, because that is where the nondeterminism was proven.

### Clone leg (2026-07-22, run `simfid-20260722-200229`) — fresh CRUD capture + one finding

The six headless CRUD cases drove cleanly on the real app (all `op exit=0`, real before/after rows; 6 normalized clone deltas ingested). > **⚠️ FINDING — `todo-reopen`: MATCH → TOLERATED(wallclock-bucket) under fresh capture. A SIMFID sim-side FIXTURE gap, masked by an over-broad tolerance. NOT a new app behaviour; NOT a proven `simulator.ts` applier bug.**
>
> The fresh clone capture's reopen delta clears `stopDate` (`date:2026-07-05 → null`) alongside `status 3 → 0` — exactly what the app does and what the case is even titled for (`todo.reopen (clears stopDate)`). The **sim-side replay delta contains no `stopDate` change at all**, so the two deltas differ on `stopDate` (sim `—` / app `null`). The `wallclock-bucket` tolerance — whose `applies()` returns true for ANY `stopDate` diff — absorbs it, so the suite reports TOLERATED rather than DIVERGENT.
>
> **Root cause (traced):** the reopen case seeds its completed pre-state with `seedTodo(db, { title: "SF Reopen", status: "completed" })`, and `insertTask` defaults `stopDate` to `null` (`test/fixtures/seed.ts:67`, `opts.stopDate ?? null`). So the sim's "completed" fixture is a completed row with **no stop timestamp** — the reopen applier has nothing to clear, and the sim delta never exhibits the `stopDate → null` transition. The **real app always stamps `stopDate` on completion** (the guest driver completes via the CLI, getting a real `2026-07-05` stamp), so its reopen genuinely clears it. The comparison is sound and sensitive — `todo-complete`/`todo-cancel` both bucket `stopDate` null → `date:2026-07-05` identically on both sides and MATCH (the host sim replay is clock-pinned to the same 2026-07-05), which is precisely why the reopen fixture's missing stopDate stands out.
>
> **Classification:** SIMFID harness fidelity gap (the synthetic completed pre-state omits `stopDate`, unlike any real completed row), compounded by an over-broad tolerance (`wallclock-bucket` should absorb bucket-VALUE jitter between two present dates, not a present-vs-`null` structural difference). It is NOT new app behaviour, and it does NOT prove the `simulator.ts` reopen applier fails to clear `stopDate` — that path is simply never exercised because the fixture provides no stopDate to clear.
>
> **Follow-ups (NOT done in that PR — per its scope, no fixture/tolerance/applier edits):** (1) seed `stopDate` on the reopen case's completed fixture so the sim reproduces a real completed pre-state and actually exercises stopDate-clearing; then re-run the clone leg to confirm MATCH. (2) Consider tightening `wallclock-bucket` so a present-vs-`null` `stopDate` difference surfaces as DIVERGENT while genuine same-day bucket jitter stays tolerated.

> **✅ RESOLVED (2026-07-22, this change) — both follow-ups landed.** (1) The reopen case now seeds a real completion `stopDate` (`lab/simfid/cases.ts`, `Date.UTC(2026,6,5,…)` on the pinned RSIM day), so the sim replay delta exhibits the same `stopDate date:2026-07-05 → null` transition the app stamps — the reopen applier's stopDate-clearing (`src/write/vectors/simulator.ts` `todo.reopen` → `setStatus(…, null)`) is now actually exercised. **Applier verification: the reopen applier was already correct** — it clears `stopDate → null`; the fixture simply never gave it a stopDate to clear, so the path was untested. - **The 5 TOLERATED verdicts** are the two headline recurrence nondeterminisms, and their presence is the point: they prove the tolerances FIRE (present-or-absent both pass) rather than mis-reporting a bug. `instance-next-sentinel` fires wherever a top-level instance is minted (fixed to-do/project + subtree cases); `rt1-child-backlink` additionally fires on the subtree cases (one instance-side child modelled as stamped, per RSIM-S's nondeterministic draw). (`wallclock-bucket` was the SIXTH TOLERATED under the raw clone leg on `todo-reopen`, but it was MASKING a sim-side fixture gap rather than a real nondeterminism — now fixed: the fixture seeds a real stopDate and the tolerance no longer absorbs a present-vs-`null` difference, so the row is an honest MATCH. The remaining CRUD rows (`suite-evidence`) and the recurrence/subtree family (`rsim-evidence`) are still banked-derived — an UNDOCUMENTED simulator divergence there would not surface until they too are clone-captured (CRUD: follow-up 1; recurrence: needs the AX-capable driver).
