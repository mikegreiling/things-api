# Novel working paths — the surprising capabilities the lab uncovered

The capabilities that exist NOWHERE in Cultured Code's documentation, discovered by probing. Each is lab-validated with the evidence id shown; the living [capability-matrix](../capability-matrix.md) tracks which are wired into shipped operations. Ordered roughly by how much product surface each unlocked.

## AppleScript

1. **`_private_experimental_ reorder to dos in <specifier> with ids "…"`** — the undocumented sdef command behind every native reorder scope. Accepts to-dos in `list "Today"` (A50) and `list "Inbox"` (21b A6, reversed wire list), un-headed children in project/area specifiers (O04/O05/O09/O10), **projects** in an area specifier (O14), **heading rows** in a project specifier (scf P1 — children follow their FK), and `list "Someday"` members (P6h) with a distinct anchor-stack model: the call's original top item never moves, to-dos stack ASCENDING (P8b two-call protocol), someday PROJECTS stack DESCENDING (P9e inverted protocol). Gated behind `allow-experimental` + the sdef canary.
2. **By-id addressing reaches rows list reads can't see.** Repeating templates (A12) AND heading rows (P10 — `to do id "<heading-uuid>"` resolves type=2 despite "no heading class") are fully addressable: `get properties`, `set name` (rename, P10d), `set status` (archive/unarchive, P10d/P10b — cascade semantics P11c/P11d). The entire shipped heading lifecycle rides this.
3. **`_private_experimental_ json of to do id …`** (A51) — full JSON document exposing checklist items and recurrence config invisible to every public read surface.
4. **Backdating existing items**: `set completion date` / `set creation date of to do id X` rewrite history (scf2 P4b) — dead on URL (oddity 2g) and Shortcuts (P4a). Ships as `todo.backdate`.
5. **`delete <property>` as the un-setter**: `delete parent tag of tag X` un-nests to root (P29); `delete keyboard shortcut of tag` clears the shortcut (21b A4) — where `set … to missing value`/`""` error out.
6. **`schedule to do id <PROJECT>` works** (P14-A3 — projects inherit the `to do` class); an unwired alternative to `update-project?when=` (decision: document-only, roadmap §D). Also `schedule … for (current date)+N*days` fills the Upcoming gap (A21B).
7. **`move <trashed to-do> to list "Inbox"` restores from Trash** (E15 — the UI's "Put Back", scriptable); `move <trashed project> to list "Anytime"` restores a project IN PLACE, restoration-faithful (P06/P07).

## URL scheme / TJSON

8. **The empty-parameter clear pattern**: `update?list-id=` detaches a to-do from its project/area (P21/P22); `update-project?area-id=` detaches a project (P24) — the ONLY container-clear on any surface (four-behaviors-for-one-intent, oddity 5g). Empty `tags=` / `checklist-items=` clear likewise (P14/P15).
9. **TJSON `{"type":"heading"}` items inside a NEW project's `items` create real type=2 rows** (HX0; used by the golden seeder and the e2e heading fixtures). The ONLY headless heading-create — but only at project creation; every relocation/append shape is dead (HX1–HX4b).
10. **At-creation backdating via TJSON attributes**: `{"completed":true,"creation-date":…,"completion-date":…}` honored exactly (scf2 P4d) — the logbook-import path. Ships as `todo.add-logged`.
11. **`update?duplicate=true` copies a to-do or a project INCLUDING children** (E07/E17) — AppleScript `duplicate` is refused (-1717).
12. **The `when=` bounce family**: front-insert on attach/round-trip powers the evening reorder (O08) and top-level sidebar-project ordering via a someday↔anytime round-trip (P7c/P8e) — the only write path to sidebar project order.

## Shortcuts (App Intents)

13. **`Create Heading` makes a heading in an EXISTING project** (S02) — the marquee capability, dead on both other vectors; exhaustively confirmed unique by the HX sweep.
14. **Single-item permanent delete** (S-delperm — `Delete` + Delete Immediately: row gone, no tombstone) — interactive-only (delete-class consent has no Always-Allow, oddity 5j).
15. **`Edit Items → Reminder Time ""` clears a DATED reminder IN PLACE** (scf P3b; works on projects too — RC avenue D) — the only *schedule-preserving* dated-reminder clear (`startDate` untouched, headless); it cannot SET (P3a). **NOT the only clear (corrected 2026-07-11, RC campaign):** a pure-URL bounce `when=today`→re-date also clears (RC01/RC02, non-atomic + transient Today re-date + crashes on repeating items), and AppleScript `move … to list "Inbox"` drops it by de-scheduling (RC03, loses the date). AppleScript has no writable reminder property — sdef exposes only read-only `activation date` + read-only `_private_experimental_ json`, `set` → −10006 (RC avenue C). See [rc-suite-results.md](../lab/rc-suite-results.md).
16. **Output-class consent is inherited by VM clones** (scf) — grant Always-Allow once in the golden, every clone runs proxies headless.
17. **The proxies themselves are extractable and re-signable**: workflow blobs live verbatim in `Shortcuts.sqlite` (`ZSHORTCUTACTIONS.ZDATA`), wrap into old-format plists, and `shortcuts sign --mode anyone` mints importable signed files (SX2–SX4) — the distribution path for `shortcuts/*.shortcut` (no iCloud, no manual rebuild).
18. **Programmatic shortcut AUTHORING — no golden GUI sitting needed** (SX5, 2026-07-10). A hand-composed (never-GUI-touched) `WFActionParameterFilterTemplates` predicate runs CORRECTLY: the SX5 repair of `find-items` was written in Python (plistlib), injected, and returned real case-folded matches. Combined with #17, new proxies can be minted entirely in code: compose the `WFWorkflowActions` bplist → wrap in the old-format envelope → `shortcuts sign --mode anyone`. The action vocabulary comes free from the app bundle: `Things3.app/Contents/Frameworks/ThingsCommon.framework/…/Metadata.appintents/extract.actionsdata` lists every entity property identifier + per-property query comparators (that's how `title` was confirmed and `name` ruled out, host-side, before any VM run). Two supporting facts: **DB surgery on a consented shortcut preserves its consent** (swap `ZSHORTCUTACTIONS.ZDATA` after killing `siriactionsd`; identity, not content, keys the grant — the SX5 iteration loop), and predicate `Property`/`Operator` values must come from the metadata (unknown ones crash Things — oddity C4).

## UI (VNC synthetic input — dedicated-Mac "ui" vector)

22. **`File → New Repeating To-Do` is VNC-drivable end-to-end** (UI1, §E½ feasibility) — the ONLY working repeat-create path on any surface (URL/AS/Shortcuts all dead). `tart run --vnc-experimental` + vncdotool posts hardware-level HID (no TCC), so menu clicks, dialog interaction, checkbox toggles, and **keyboard entry (titles AND numeric offset fields)** all work despite Accessibility being ungranted (osascript System Events → −1719, so there is NO AppleScript UI-scripting fallback). Layout is deterministic at the golden's 2048×1536 framebuffer. Green-lights a dedicated-Mac "ui" vector for the conclusively-UI-only surface (repeat create/edit, sidebar order, to-do↔project convert). VNC-keyboard gotchas: `vncdo type` sends letters lowercase; `super-a` types a literal 'a' (no Cmd) — clear a field with N×bsp + N×del, never select-all.
23. **The deadline-ness of a repeat lives in the TEMPLATE's `deadline` column, not the rule** (UI1) — NULL = deadline-less (the GUI default), `4001-01-01` sentinel = deadlined. The rule blob is byte-identical for deadlined-vs-deadline-less ts=0; a non-zero start offset also surfaces as `ts=−N`. Universal across fixed AND after-completion. Fixed the phantom-deadline projection (`RepeatingInfo.deadlined`; oddities §8a).
24. **System Events Accessibility scripting drives Things transforms by element NAME — in a VM, SIP enabled** (AXVM1, 2026-07-14). A one-time **user-path TCC toggle** (System Settings → `tccd`; the denied AX attempt auto-creates a toggleable `sshd-keygen-wrapper` row, flip 0→2 — NOT a direct DB write, which SX4 proved read-only under SIP) makes `osascript … System Events` UI-scripting work over SSH. Recipe: select the target by stable handle `things:///show?id=<uuid>` (Things list rows are NOT AX-addressable — sparse rendering), then `click menu item "Pause" of menu 1 of menu item "Repeat" of menu "Items" of menu bar item "Items" of menu bar 1` → `rt1_instanceCreationPaused` 0→1. This is a SECOND ui-vector driving path alongside VNC, and strictly better for a headless host: **no frontmost app required, NO focus steal** (paused a template with Finder frontmost), and **works under a locked session** (VNC coordinate clicks do neither). Grant persists across reboot. Discriminator gotcha: `get name of first process` is NOT AX-gated — use a UI-element read to detect the −1719 gate. See [axvm1-accessibility.md](../lab/axvm1-accessibility.md).
25. **Synthetic mouse input from pure JXA — no compiled helper — opens the card and drags the sidebar** (NATIVE1, 2026-07-15). `osascript -l JavaScript` + the ObjC bridge resolves any AX element's frame (`AXUIElementCopyAttributeValue` for `AXPosition`/`AXSize`, read via `ObjC.castRefToObject` + `CFCopyDescription`-regex; attribute names must be NSStrings — `$('AXPosition')`) and synthesizes clicks/drags (`CGEventCreateMouseEvent`). **A double-click at an AX-resolved list-row center opens the to-do card** (the card-only surface UI2-i/UIC1 needed — the repeat bar becomes reachable, unblocking `todo.stop-repeat`); **a mouse-down → interpolated `kCGEventLeftMouseDragged` → mouse-up between two AX-resolved sidebar rows reorders an AREA** (unblocking the LAST unautomatable ordering, matrix line 107) and writes **`TMArea."index"`** (0 → −448; sparse/negative, lower sorts higher — the column O13 predicted the GUI writes only on drag). **Critical: the events must go through `CGEventPost(kCGHIDEventTap, …)`; `CGEventPostToPid` is inert** for Things' hit-testing. So this path is **foreground-bound** (no background, no under-lock — unlike the AX menu path #24) and needs **only** the Accessibility grant (CGEvent posting is Accessibility-authorized; no Input-Monitoring/PostEvent class). See [native1-spike.md](../lab/native1-spike.md).
26. **Sidebar entities ARE AX-addressable by description** (NATIVE1) — a refinement of "Things list rows aren't AX-addressable" (#24, UIC1): that holds for the *main list* (rows expose only images, no title/actions), but **sidebar area/project rows carry an `AXDescription`** (`d="LAB-AREA-A."`) so they're locatable by name + frame, and **`AXSelectedRows` READS correctly** on both tables (resolves the `things:///show`-selected row for a double-click target). Rows still expose **zero AX actions** (empty action list; `AXScrollToVisible` → −25206), so *setting* selection / scrolling stays mouse-synthesis territory.
27. **Scroll-then-drag recipe for off-viewport sidebar targets — with a computable scroll distance** (AXDRAG1, 2026-07-15). Because `AXScrollToVisible` is dead on rows (#26), off-viewport drop targets are reached with **synthesized scroll-wheel events**: move the pointer over the sidebar, then `CGEventCreateScrollWheelEvent($(), kCGScrollEventUnitLine, 1, ±delta)` → `CGEventPost(kCGHIDEventTap)` (~30px content travel per 3-line click; negative reveals lower rows). Crucially the sidebar `AXScrollArea`'s child `AXScrollBar` (`AXVerticalOrientation`) **exposes `AXValue` as a Float64 fraction 0.0→1.0** (0 top, +1.0 bottom) — so scroll distance is *computable* from the target row's virtual frame + the viewport rect, not just scroll-and-check. **Things also auto-scrolls mid-drag** when the drag hovers near the top/bottom edge (bidirectional, runs to the range end) — but it is time-based/imprecise, so a deterministic op should pre-scroll the target into the visible band, re-resolve frames, then drag within one viewport. See [axdrag1-reorder.md](../lab/axdrag1-reorder.md).
28. **Drag-drop geometry: lifting the source collapses its slot (the "downward −1 slot" rule) + drop-centre-vs-boundary semantics** (AXDRAG1, 2026-07-15). Sidebar area rows interleave with h=16 spacer rows → **40px per slot**. When a row is picked up, everything below its original position shifts **up by one slot (40px)**, so **downward** drops must target `neighborStaticY − 40` while **upward** drops use static coordinates (targets above the source don't move). Insert-before/after follows the pointer's row-half against the live layout. In the **Tags window** the same drag doubles as re-parent: **dropping on a row CENTRE nests (`TMTag.parent`), dropping on a row BOUNDARY reorders (`TMTag."index"`)** — a mis-aimed reorder silently nests. Both `TMArea` and `TMTag` reorders use a sparse/negative index (lower sorts higher) and can renumber a *neighbour* rather than the moved row (so undo captures the full order, not one index). See [axdrag1-reorder.md](../lab/axdrag1-reorder.md).
29. **Repeating-PROJECT ops via a HYBRID synthetic-click + AX path** (UIC2, 2026-07-15). A project's title-adjacent **`…` menu** carries **Repeat…** (the Items menu bar does NOT, for a shown project), and it opens the **same Repeat dialog sheet as a to-do** (byte-identical controls: freq pop-up `_NS:116`, OK `_NS:164`). So all four project repeat ops work: `make-repeating` (new template `fu`=256 + spawned instance, identity replacement), `reschedule` (repeat-bar ▸ Change…, `fu` 256→8, identity preserved), `pause`/`resume` (repeat-bar ▸ Pause↔Resume, `rt1_instanceCreationPaused` toggle). A repeating project exposes an **always-visible repeat bar** in its view (no card double-click — the to-do repeat bar is card-only). BUT this is **not pure-AX**: the `…` button is not an AX node, and the `…`/repeat-bar popover items are AX-readable-but-**inert-to-`AXPress`** — opening/navigating them needs a synthetic MOUSE click (their `AXPosition` IS resolvable, so synthesis can avoid guessed pixels); only the dialog sheet is AX-actuatable (and backgroundable, no focus steal). `open -g things:///show?id=<project>` reveals+selects a project WITHOUT activating Things. See [uic2-project-menu.md](../lab/uic2-project-menu.md).
30. **The project repeat-bar POPOVER is a separate `AXUnknown` top-level window** (UIC3, 2026-07-15) — a single HID click at the AX-resolved repeat-bar frame center (`text area 2` of the header cell) opens the [Change… · Pause↔Resume · Stop · Show Latest] popover **not** as `pop over 1` of the standard window but as a **new ≈215×220 `AXUnknown` window** (same custom-window shape as the `…` menu). Two `AXUnknown` windows coexist while it's open (the popover + the hidden 40×40 utility window), so it's addressed by subrole **and** `size is not {40, 40}`; its items are `UI element`s of the window's `scroll area 1`, matched by `AXDescription`, clicked at their AX-resolved frames (`AXPress` is inert). This is the addressing that made **`project.reschedule-repeat`/`pause-repeat`/`resume-repeat` lab-certifiable through the production CLI** (UIC3-b), on the new `click-element` mouse-synthesis driver primitive. See [uic3-build-certification.md](../lab/uic3-build-certification.md).
31. **`AXSelectedRows` is SETTABLE by pure AX → a project selects with ZERO mouse input, and `project.make-repeating` is 100% pure-AX** (UIC4, 2026-07-15 — **corrects UIC3-a**). A project **does** render as a **selectable content row** at the top of its area's (and the Someday view's) content `AXTable` — the row exposes only images (no title text), but `AXUIElementSetAttributeValue(contentTable, "AXSelectedRows", NSArray[projectRow])` returns `kAXErrorSuccess` and the selection **lands** (`AXUIElementIsAttributeSettable`→YES; NATIVE1/AXVM1 only proved *reads*). With the project so selected, **`Items ▸ Repeat…` is present and enabled**, so `select-row (AX) → Items ▸ Repeat… (AX) → Repeat sheet (AX)` drives make-repeating **100% pure-AX and background-capable with no focus steal** (DB-verified: identity replacement, new template `fu`=256 + spawned instance, **area preserved**). This is a *better* profile than the reschedule/pause/resume ops (which still need mouse-synthesis for the `…`/repeat-bar popover). Wrinkle: **backgrounded, the Repeat editor detaches to a top-level `AXUnknown` window** (address by `subrole AXUnknown, size ≠ {40,40}`; controls are direct children — `pop up button 1 of window 1` = frequency) rather than an attached `AXSheet`. See [uic4-project-selection.md](../lab/uic4-project-selection.md).
32. **A project's right-click context menu is a real `NSMenu` carrying Repeat… — but AX-opaque** (UIC4, 2026-07-15). A synthetic HID right-click on a project **row** (area view) or an area-less project **header** (Anytime view, the "Image 22" pattern) opens a genuine native `NSMenu` (system styling, `Services ▸` submenu, `Repeat…` present). Unlike the custom `…` menu (#29, AX-readable-but-inert), this one is **not published to the AX tree at all**: while open, a JXA walk of the app's children shows no `AXMenu` node, `menu 1 of window 1` → `-1728`, and `entire contents` **hangs** (the modal tracking loop blocks synchronous AX). So it's **not `AXPress`-drivable** — the only drive is **keyboard typeahead** (`keystroke "repeat"` + Return, which the tracking loop consumes), English-title-pinned and foreground-bound. It's the in-place opener for the area-less-anytime case (which has no selectable row). See [uic4-project-selection.md](../lab/uic4-project-selection.md).
33. **`make-repeating` normalizes a project's `start` to the repeating bucket + templates resist quiet-vector reschedule** (UIC4, 2026-07-15). Making any project repeat writes `start`=2 to the template AND its spawned instance **regardless of the project's prior anytime/someday state** (even the app-made seed is `start`=2; the instance carries a `startDate` → shows in Upcoming). Consequence: **area is preserved through the conversion, `start` is NOT** — so a temporary Someday-coercion of an area-less anytime project (via the `update-project?when=someday&auth-token` quiet vector, to make it row-selectable) is **cleanup-free**, since the conversion discards the prior bucket anyway. Separately, a repeating **template refuses quiet-vector schedule edits**: `things:///update-project?when=…` is a silent no-op on its `start`, and AppleScript `move … to list "Anytime"` → **error 301** (re-homing a template's schedule needs the in-app UI). See [uic4-project-selection.md](../lab/uic4-project-selection.md).

## Reads / environment

19. **`uriSchemeEnabled` on disk** (21b) — the group-container preferences plist carries the true "Enable Things URLs" state (the auth token does NOT track it); powers the availability layer + failure attribution.
20. **The Today comparator** (`startBucket, todayIndexReferenceDate DESC, todayIndex, uuid` + the deadline-pull membership rule with `deadlineSuppressionDate` re-arming) — UI-oracle-derived, exact 393/393 live reconciliation (today-order-research).
21. **Hidden lists** `list id "tomorrow"` and `list id "later-projects"` exist in the sdef enumeration (P9a) — unprobed as reorder specifiers (backlog).
22. **The Upcoming deadline-forecast rule** (UPC1) — a future deadline forecasts a `startDate IS NULL AND start IN (1,2)` row (anytime/someday to-dos AND someday projects) into GUI Upcoming under the **deadline** date, unless nag-suppressed; **Inbox (start=0) is excluded**. Scheduled rows still group by when-date; a when+deadline row is one row under its when-date. Group/sort key `COALESCE(startDate, deadline)`. Mirrors the Today deadline-pull one step earlier — the CLI `upcomingView` fix (upcoming-research.md).
23. **`deadlineSuppressionDate` is set by rescheduling an overdue item to Someday/Anytime** (UPC1, a WRITE path via URL `update`/`update-project when=someday|anytime`) — the only way to stamp the suppression the todayView predicate reads; no dismiss command exists (oddities §8e). Rescheduling to a future DATE relies on `startDate` instead (no stamp).
