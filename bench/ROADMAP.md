- 2026-07-17: **Metrics anomaly to investigate before round 1**: MCP-arm median `tokensIn` (2.3k) is far BELOW its measured static context (16.6k tool catalog) — the codex backend's usage.input likely reports cache-discounted tokens, so `tokensIn` under-counts true context on cache-friendly arms. - 2026-07-17: **HOST-ESCAPE INCIDENT + quadruple-fence response.** During Phase 0, a scaffolding-worktree smoke ran the write sample on a branch that PREDATED the fence: `THINGS_SIM_WRITES=1` was inert there, `defaultVectors` returned real transports, and 4 real `things:///add?title=Water%20the%20plants` dispatches reached the operator's live app (each verify-failed against the fixture, masking the escape as a benign FAIL); a fixture-uuid reveal also popped a modal, and bench audit records polluted the real audit trail via wrong env names (`THINGS_STATE_DIR` vs the real `THINGS_API_STATE_DIR`). Cleanup: audit scrubbed with backup; the 4 synthetic todos left for Mike's manual deletion (prod writes are forbidden even for cleanup). Hardening (all shipped, all tested): (1) `defaultVectors` marker backstop — a `benchFixture`-marked DB (env or client-opened) can NEVER be paired with real transports, throws without the full fence; marker made opt-in in `buildFixtureDb` so ordinary unit fixtures still exercise real-vector seams; (2) fence requires scratch `THINGS_API_STATE_DIR`/`THINGS_API_CONFIG_DIR`; (3) reveal gate keys on the opened DB's marker, not just env; (4) runner zero-dispatch preflight — a `--dry-run` canary must compile to `simulated:todo.add` before any task runs; (5) schema tripwire — `SIMULATED_DATABASE_VERSION` pin refuses drifted fixtures until the modeling chain is re-verified (drift-runbook step 5). Standing lesson: never run bench smokes from a worktree whose base predates the current fence — the preflight now makes that impossible to miss.
